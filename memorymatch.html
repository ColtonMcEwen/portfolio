Memory Matching

Web Application

This was a fun, but challenging project I did at school. It was challenging because I had to allow users to add, delete, and edit data from a database managed in Python. On top of that, the data had to be specific to a user. That meant I had to enable users to register and/or log in. Though even this wasn't enough, I needed to develop an interactive way for a user to use the web application. Thus came the idea to make a memory matching game.

People have likely played this at a young age. A grid of cards is laid out in front of the individual faced-down. The goal is to flip exactly two cards over and check if they match. If they don't, flip both cards back facing down. Repeat this with any combination of cards until two cards match. From there, keep the matching cards faced up and repeat with other cards until eventually, all the cards have matched.

The game keeps count of all attempts to match any two cards until all have matched. Once finished, the score is written to the database and is added to a list of other player scores!
